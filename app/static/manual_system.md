# 5 Way House 系統管理手冊

> **文件版本**：v1.0 &emsp; **最後更新**：2026-03-22  
> 本手冊涵蓋全域系統設定、帳號建置、資料備份與伺服器部署指南，適用於 `Admin` 與 `Manager` 級別管理員。

---

## 1. 系統網址與存取

**系統網址**：`http://<您的伺服器IP>:5001`

**權限說明**：
- **Admin（管理員）**：系統最高權限，可存取所有設定與日結補登功能。
- **Manager（經理）**：進階管理權限，可管理據點、修改商品類別、檢視全域報表。

---

## 2. 管理員基礎功能

管理員功能位於系統任一模組上方導覽列右側的「**系統後台管理**」下拉選單。

### 2.1 據點管理

**路徑**：系統後台管理 → 據點管理

管理所有營業據點，包含：
- **新增據點**：填寫名稱和 Slug（網址代碼，英文小寫加連字號，例如 `main-store`）
- **編輯據點**：修改名稱或 Slug
- **刪除據點**：刪除未有任何交易記錄的據點

> ⚠️ **注意**：Slug 需全球唯一，且修改後已分享的連結將失效。

### 2.2 使用者管理

**路徑**：系統後台管理 → 使用者管理

管理所有系統使用者帳號：
- **新增使用者**：設定使用者名稱、密碼與針對模組的角色分配。
- **編輯使用者**：修改密碼或重新指派角色。
- **刪除使用者**：移除離職員工。

> 💡 如需允許使用者以 Google 帳號登入，需在「編輯」頁面填入對應的 Google 帳號電子郵件。

### 2.3 角色與權限設定

**路徑**：系統後台管理 → 角色與權限設定

系統中各個模組具備極度精密的「細粒度角色權限 (Fine-Grained RBAC)」與「據點隔離屏障 (Location-Binding)」：

**新建或編輯角色的控制面板（全視覺化勾選）：**
1. **系統功能模組授權**：
   - 權限不再是過往粗劣的「管理員」與「平民」二分法。現在細分出諸如 **「收銀台操作」、「編輯補登日結」、「合併報表總結算」** 等十幾種細緻開關。
   - 總部能精確發放特製權限配置（例如：建立一個僅能查閱報表但無權操作收銀機的「財務總機查帳員」）。
2. **實體據點存取授權 (地理屏障)**：
   - 在角色介面的右側網格中，您能勾選該角色被允許操作的「特定實體據點」。
   - 若店長帳號僅被勾選了 `壽豐店`，該員工登入後，**儀表板與報表系統將全自動隱形其他非管轄據點**。任何跨區越權的網址嘗試都會遭系統攔截阻擋。

### 2.4 商品類別管理

**路徑**：系統後台管理 → 據點管理 → 選擇據點 → 商品類別

每個店鋪據點可獨立設定：
- **一般商品**：計入銷售總額。
- **其他現金收入**（如捐款）：單獨列計，不計入常規銷售中。

## 3. 全域系統與雲端備份設定

**路徑**：系統後台管理 → 全域系統設定與備份（限 Admin）

設定 Google Drive 與 Google Sheets 自動雲端備份機制：

1. **雲端硬碟資料夾名稱**：備份檔案存放目錄（預設 `Cashier_System_Reports`）。
2. **授權步驟**：
   - 點按「**連結 Google 帳號**」。
   - 跳轉到 Google 授權頁面，選擇管理員 Google 帳號並允許存取。

系統將在每日「日結歸檔」時自動透過 Redis 工作隊列上傳當日對帳報表。若需手動備份，點按「**手動立即執行雲端備份**」按鈕發送指令即可。

---

## 4. 補登日結（強制結帳）

當某個據點因特殊情況（如收銀員下班忘記關機）而未完成當日結帳，管理員可由後台啟動補登。
1. 在報表系統中找到異常的 `OPEN` 紀錄。
2. 進行強制「**補登日結**」。
3. 人工填寫當日盤點數字並代理簽署完成帳務封存。

## 5. 系統伺服器部署指南

### 5.1 首次部署指令

```bash
# 1. 取得專案
git clone <repository-url>
cd 5_way_house

# 2. 複製並設定環境變數
cp .env.example .env

# 3. 啟動服務 (自動化建立 Database 與 Phase 2 Default User Accounts)
docker-compose --env-file .env up -d --build
```

### 5.2 環境變數說明 (`.env`)

| 變數 | 說明 |
|------|------|
| `SECRET_KEY` | Flask 加密金鑰 |
| `DATABASE_URL` | 預設為 SQLite。可改換為 PostgreSQL 連線字串 |
| `REDIS_URL` | 預設 `redis://redis:6379/0` |

### 5.3 備份還原

```bash
# 還原 SQLite 實體備份庫
cp instance/app.db.backup instance/app.db
docker compose restart web
```

### 5.4 Google API 憑證掛載
為啟用 Google 雲端備份與 Google 登入功能，必須建立 GCP 專案並配置憑證：
1. 向 GCP 申請 Web App OAuth 2.0 用戶端 ID 憑證 JSON 檔，下載並重新命名為 `client_secret.json`。
2. 將檔案透過 FTP/SCP 放置於伺服器部署根目錄內的 `instance/` 目錄內。
3. 若伺服器沒有對外開放網域來承接 Google 的 Callback URL，您需要：
   - 先在**本地開發機**的 `config.py` 環境中放入 `client_secret.json` 並執行一次系統的 Google 登入認證，這將自動產生 `token.json`。
   - 將產生的 `token.json` 一併手動上傳至伺服器 `instance/` 目錄。
   - 伺服器後續透過這份 token 即可進行背景操作而無需再次開啟網頁授權。
