## 1. 系統簡介

**5 Way House 收銀系統**是一套專為多據點零售業設計的 Web 收銀與報表管理系統，主要功能包括：

- 多據點獨立開帳、收銀、結帳
- 即時銷售統計（交易筆數、商品件數、銷售金額）
- 每日合併報表結算（多據點彙總）
- 銀行存款收據 OCR 自動比對
- 雲端備份（Google Drive / Google Sheets）
- PDF 報表列印匯出
- **（下一階段開發中）** 倉庫物流、工坊物資登錄、住宿時段管理、志工證書生成系統

**系統網址**：`http://<您的伺服器IP>:5001`

## 2. 角色與權限

| 角色 | 說明 | 可存取功能 |
|------|------|------------|
| **Admin（管理員）** | 系統最高權限 | 所有功能，包含管理後台、報表查詢與結算 |
| **Cashier（收銀員）** | 日常操作 | 儀表板、開帳、收銀、結帳（簽名、確認報表）|

> 💡 預設管理員帳號：`admin`（請於首次登入後立即修改密碼）

## 3. 登入與登出

### 3.1 登入與系統首頁 (Portal)

1. 前往系統主頁 `/cashier/login`
2. 輸入**使用者名稱**與**密碼**
3. 點按「**登入**」

> 系統亦支援 **Google 帳號登入**（需管理員預先設定 OAuth），一旦綁定 Google 帳號，點按「以 Google 登入」即可免密登入。

成功登入後，您會切換至**五角拌營運入口首頁 (Portal)**。此頁面採用大區塊版塊設計：
- **實體據點營運**：點擊卡片可快速進入「POS 收銀終端」、「營業報表與管理卡」或「系統後台設定」。
- **下階段開發計畫**：首頁下方列有倉庫、工坊、住宿與志工系統等模組，目前仍處於開發中狀態。使用者後續不僅可以利用左上方導覽列切換，也可以隨時點選系統左上角的 Logo 返回此首頁入口重新跳轉。

### 3.2 登出

- 點按右上角導覽列「**登出（使用者名稱）**」即可安全登出。

## 4. 每日開帳

**路徑**：儀表板 → 選擇據點 → 點按「開始本日營業」

### 步驟

1. 在儀表板確認據點狀態為「**尚未開帳**」
2. 點按據點進入「**開帳表單**」
3. 填寫「**開店準備金**」（即當日零錢備用金，例如 $500）
4. 點按「**確認開帳**」

> ⚠️ **注意**：每個據點每天只能開帳一次。若需要在已關帳的日期補登，請使用「補登日結」功能（需管理員權限）。

## 5. 收銀操作（POS）

**路徑**：儀表板 → 選擇「營業中」的據點

### 5.1 介面說明

| 區域 | 說明 |
|------|------|
| **商品類別按鈕** | 點按快速選擇商品類別與單價 |
| **交易清單** | 顯示本次尚未結帳的商品明細 |
| **折扣欄位** | 可輸入單筆折扣金額（正整數）|
| **總金額** | 自動計算（含折扣）|

### 5.2 操作流程

1. 點按**商品類別**按鈕（或直接輸入金額）
2. 調整**數量**（預設為 1）
3. 如需折扣，在折扣欄位輸入折扣金額
4. 確認總金額後，點按「**結帳**」

> 💡 收銀畫面支援**鍵盤快速操作**，適合桌面使用。行動裝置可直接點按觸控按鈕。

### 5.3 商品類別說明

- **一般商品**：計入銷售總額
- **其他現金收入**（如捐款）：單獨列計，不計入常規銷售

## 6. 每日結帳

**路徑**：儀表板 → 選擇「待確認報表」的據點

### 6.1 結帳前準備

確認所有交易均已輸入完畢後，進入「**關帳**」頁面。

### 6.2 現金盤點

1. 在現金盤點表格中填入各**面額紙幣與硬幣數量**
   - 面額：$1000、$500、$200、$100、$50、$10、$5、$1
2. 系統自動計算「**盤點現金合計**」

### 6.3 簽名確認

1. 在簽名欄分別由「**作業員**」、「**覆核員**」、「**出納員**」完成電子簽名
2. 點按「**確認結帳並提交報表**」

### 6.4 報表確認

系統跳轉至「**每日報表**」頁面，顯示：

| 欄位 | 說明 |
|------|------|
| 開店準備金 | 當日零錢備用金 |
| 本日銷售總額 | 所有交易合計 |
| 帳面總額 (A) | 準備金 + 銷售額 |
| 盤點現金合計 (B) | 實際盤點結果 |
| 帳差 (B - A) | 正數為溢收，負數為短收 |
| 交易筆數 | 當日顧客數 |
| 銷售件數 | 當日商品件數 |

確認無誤後，點按「**確認並歸檔報表**」完成結帳。

## 7. 銀行存款收據驗證

**路徑**：報表 → 合併報表結算 → 銀行存款收據驗證

結帳後，員工需將現金存入銀行，拍攝存款收據並上傳，系統將自動 OCR 識別金額並與帳面存款數據比對。

### 操作步驟

1. 完成合併結算頁（確認存款欄 H 的金額）
2. 在「**銀行存款收據驗證**」卡片區塊：
   - 點按「**選擇檔案**」或直接用手機**拍照**上傳存款收據
3. 點按「**掃描並驗證**」
4. 系統顯示比對結果：
   - **✅ 金額一致**：收據金額與系統記錄相符
   - **⚠️ 金額不符**：顯示差異金額，請核查後確認
5. 點按「**確認一致，儲存**」或「**確認差異，仍儲存**」完成記錄

> 💡 支援圖片格式：JPG、PNG、WEBP。行動裝置可直接調用相機拍照。

> ⚠️ OCR 識別精度受照片清晰度影響。若光線不足或角度傾斜，可能無法自動識別，請人工核查後手動確認。

## 9. 合併報表結算

**路徑**：導覽列 → 合併報表結算（需管理員權限）

每日所有據點結帳完成後，管理員進行**合併結算**，彙總各據點數據。

### 財務欄位說明

| 欄位 | 說明 |
|------|------|
| A | 應有現金 = B + C + D |
| B | 開店現金（各據點準備金合計）|
| C | 手帳營收（銷售合計）|
| D | 其他現金（捐款等）|
| E | 實有現金（各據點盤點合計）|
| F | 溢短收 = E - A |
| H | 存款（E 減去明日開店現金）|
| I | 明日開店現金（手動填入）|

### 結算步驟

1. 選擇結算日期
2. 確認所有據點均已結帳（否則無法結算）
3. 填入「明日開店現金 (I)」，系統自動計算存款 (H)
4. 填寫各欄備註（選填）
5. 點按「**儲存結算**」
6. 上傳銀行存款收據（參見第 7 章）
7. 點按「**下載 PDF**」列印歸檔

## 10. 管理員功能

管理員功能位於導覽列右側「**管理**」下拉選單。

### 10.1 據點管理

**路徑**：管理 → 據點管理

管理所有營業據點，包含：
- **新增據點**：填寫名稱和 Slug（網址代碼，英文小寫加連字號，例如 `main-store`）
- **編輯據點**：修改名稱或 Slug
- **刪除據點**：刪除未有任何交易記錄的據點

> ⚠️ **注意**：Slug 需全球唯一，且修改後已分享的連結將失效。

### 10.2 使用者管理

**路徑**：管理 → 使用者管理

管理所有系統使用者帳號：
- **新增使用者**：設定使用者名稱、密碼與角色
- **編輯使用者**：修改密碼或重新指派角色
- **刪除使用者**：移除不需要的帳號

> 💡 如需允許使用者以 Google 帳號登入，需在「編輯」頁面填入對應的 Google 帳號電子郵件。

### 10.3 角色權限設定

**路徑**：管理 → 角色權限設定

系統預設角色：
- **Admin**：完整管理權限
- **Cashier**：日常收銀操作

管理員可新增自訂角色，並分配不同權限組合。

### 10.4 商品類別管理

**路徑**：管理 → 據點管理 → 選擇據點 → 商品類別

每個據點可獨立設定商品類別：

| 欄位 | 說明 |
|------|------|
| 類別名稱 | 顯示在 POS 收銀介面的按鈕標籤 |
| 單價 | 每件商品的預設價格 |
| 類型 | `general`（一般商品）或 `other_income`（其他現金收入）|

> 💡 類型設定為「其他現金收入」時，金額將單獨列計，不計入銷售總額。

### 10.5 系統設定（Google 整合）

**路徑**：管理 → 系統設定

設定 Google Drive 與 Google Sheets 雲端備份：

| 設定項目 | 說明 |
|----------|------|
| 雲端硬碟資料夾名稱 | 備份檔案存放的資料夾（預設 `Cashier_System_Reports`）|
| Sheets 檔案名稱格式 | 報表試算表的命名模板（支援 `{location_name}`、`{year}` 變數）|

**授權步驟**：

1. 點按「**連結 Google 帳號**」
2. 跳轉到 Google 授權頁面，選擇管理員 Google 帳號並允許存取
3. 授權成功後回到設定頁

## 11. 補登日結（強制結帳）

**路徑**：管理 → 補登日結（需管理員權限）

當某個據點因特殊情況（系統故障、操作失誤）未完成當日結帳，管理員可使用此功能補登。

### 操作步驟

1. 選擇日期與據點，篩選出「**OPEN**」狀態的未結帳記錄
2. 點按「**補登日結**」進入補登表單
3. 填入現金盤點數據，完成簽名
4. 點按「**確認補登**」

## 12. 資料備份與雲端同步

### 12.1 即時備份

在「每日報表」確認歸檔時，系統自動觸發雲端備份（透過 Redis 工作隊列非同步處理）：
- **Google Drive**：上傳 PDF 報表
- **Google Sheets**：同步當日銷售數據

### 12.2 手動備份

**路徑**：收銀員儀表板 → 系統設定 → 「**立即備份當日資料**」按鈕

若需手動觸發備份，點按此按鈕即可將今日資料即時同步至雲端。

## 14. 常見問題 FAQ

### Q1：忘記密碼怎麼辦？
請聯繫系統管理員，由管理員至「**使用者管理**」頁面重設密碼。

### Q2：POS 頁面沒有商品按鈕？
請管理員至「**商品類別管理**」為該據點新增商品類別。

### Q3：OCR 無法識別存款金額？
1. 確保照片清晰、光線充足
2. 確保收據上的金額文字未被遮擋
3. 若多次失敗，可人工核查後手動確認結果

### Q4：報表結算頁顯示「尚有據點未完成結帳」？
必須等待所有已開帳的據點完成結帳後才能進行合併結算。請確認各據點的結帳狀態。

### Q5：刪除據點後資料會消失嗎？
若據點有關聯的交易記錄，系統將禁止刪除，資料受到保護。

### Q6：雲端備份失敗怎麼辦？
1. 確認 Google 帳號授權仍然有效（前往系統設定重新授權）
2. 確認 Redis 服務正在運行（Docker 環境下執行 `docker-compose ps` 確認 worker 狀態）

## 15. 系統管理員設置指南

### 15.1 環境需求

| 需求 | 最低規格 |
|------|----------|
| 作業系統 | Ubuntu 20.04 / macOS 12+ |
| Docker | 20.10+ |
| Docker Compose | 2.0+ |
| 記憶體 | 2 GB |
| 硬碟 | 10 GB |

### 15.2 首次部署

```bash
# 1. 取得專案
git clone <repository-url>
cd 5_way_house

# 2. 複製並設定環境變數
cp .env.example .env
# 編輯 .env，設定 SECRET_KEY、資料庫等設定

# 3. 啟動服務
docker-compose --env-file .env up -d --build

# 4. 初始化資料庫
docker-compose exec web flask db upgrade

# 5. 建立管理員帳號
docker-compose exec web flask auth create-user admin <密碼> --role Admin
```

### 15.3 環境變數說明（`.env`）

| 變數 | 說明 | 範例 |
|------|------|------|
| `SECRET_KEY` | Flask Session 加密金鑰（務必修改）| `your-very-secure-key` |
| `FLASK_ENV` | 執行環境 | `production` |
| `DATABASE_URL` | 資料庫連線字串 | `sqlite:///instance/app.db` |
| `REDIS_URL` | Redis 連線字串 | `redis://redis:6379/0` |

### 15.4 備份與還原

**備份資料庫**：
```bash
# SQLite（本地開發）
cp instance/app.db instance/app.db.backup

# PostgreSQL（生產環境）
docker-compose exec db pg_dump -U <user> <dbname> > backup.sql
```

**還原資料庫**：
```bash
# SQLite
cp instance/app.db.backup instance/app.db
```

### 15.5 Google API 憑證設定

1. 前往 [Google Cloud Console](https://console.cloud.google.com/)
2. 建立專案 → 啟用「Google Drive API」與「Google Sheets API」
3. 建立「OAuth 2.0 用戶端 ID」（應用程式類型：網頁應用程式）
4. 設定授權重新導向 URI：`http://<您的網址>/google/callback`
5. 下載憑證 JSON 檔，重新命名為 `client_secret.json`，放置於 `instance/` 目錄
